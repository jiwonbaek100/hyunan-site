# 성공사례(카드뉴스) 추가 가이드 — 법률사무소 현안 홈페이지

## 1. 이 홈페이지의 성공사례는 "카드뉴스 + 네이버 블로그 연동" 방식입니다

홈페이지에는 자체 글 작성 공간이 없습니다. 대신:

1. 실제 사례 글은 **네이버 블로그(an_law)**에 작성하시고
2. 홈페이지에는 **카드뉴스(썸네일 + 제목 + 요약)** 만 등록하면
3. 방문자가 카드를 누르면 **네이버 블로그 글로 새 창에서 바로 이동**합니다.

카드 썸네일은 기본적으로 **법률사무소 현안 로고**가 분야별 색상 배경 위에 떠 있는
형태로 자동 표시됩니다. 사진을 따로 준비하지 않아도 됩니다.

---

## 2. 수정할 파일은 `index.html` 단 하나입니다

이 홈페이지는 **한 페이지(`index.html`)로만 구성**되어 있습니다.
(이전에 있던 `cases.html`, `data.js`는 더 이상 사용하지 않습니다 — 카드를 누르면
바로 블로그로 이동하므로 별도 목록 페이지가 필요 없습니다)

메모장으로 `index.html`을 열고, **`const CASES = [ ... ]`** 부분을 찾아 수정하세요.

---

## 3. 새 사례를 추가하는 순서

1. **네이버 블로그(an_law)에 성공사례 글을 먼저 작성**합니다.
2. 작성이 끝나면 그 글의 **주소(URL)를 복사**합니다.
   예) `https://blog.naver.com/an_law/224319618023`
3. `index.html`을 메모장으로 엽니다.
4. `const CASES = [` 로 시작하는 부분을 찾습니다.
5. 아래 "■ 빈 템플릿"을 복사해서, **마지막 사례의 `},` 바로 다음 줄**에 붙여넣습니다.
   - 마지막 사례의 닫는 중괄호 `}` 뒤에 쉼표(`,`)가 없다면 먼저 추가하세요.
6. 따옴표 안의 내용을 실제 사례 정보로 바꾸고, `blogUrl`에는 2번에서 복사한 주소를 넣습니다.
7. 저장 후 `index.html`을 다시 열어 "성공사례" 카드뉴스가 잘 보이는지,
   카드를 눌렀을 때 방금 쓴 블로그 글로 이동하는지 확인합니다.

### ■ 빈 템플릿 (복사해서 쓰세요)
```
{
  category: "분야를 입력하세요(부동산/형사/민사/가사 중 하나)",
  tag: "분야 · 키워드를 입력하세요",
  title: "제목을 입력하세요",
  summary: "한눈에 보이는 요약을 입력하세요",
  date: "2026-01-01",
  blogUrl: "https://blog.naver.com/an_law/여기에글번호",
  image: ""
},
```

---

## 4. 입력 항목 설명

| 항목 | 설명 | 예시 |
|---|---|---|
| `category` | 반드시 `부동산` / `형사` / `민사` / `가사` 중 하나만 | `"민사"` |
| `tag` | 카드에 표시되는 작은 분류 태그 | `"민사 · 손해배상"` |
| `title` | 카드 제목 (블로그 글 제목과 맞추는 걸 추천) | |
| `summary` | 카드에 보이는 2~3줄 요약 | |
| `date` | 글 작성일 (`YYYY-MM-DD` 형식, 최신순 정렬에 사용) | `"2026-06-17"` |
| `blogUrl` | 네이버 블로그 글 주소 (정확히, 공백 없이) | `"https://blog.naver.com/an_law/224319618023"` |
| `image` | 카드 썸네일 사진 주소. **비워두면(`""`) 현안 로고가 분야별 색상 배경 위에 자동 표시**됩니다 | `""` 또는 사진 주소 |

### 카드에 실제 사진을 쓰고 싶다면
`image` 항목에 인터넷에 올라가 있는 이미지 주소를 넣으면 됩니다.
비워두면 지금처럼 **현안 로고 + 분야별 색상**이 자동으로 카드뉴스 썸네일처럼 표시됩니다.

## 5. 의뢰인 보호를 위한 주의사항

- 카드의 `title`, `summary`, 그리고 사용하는 사진 모두에서 실제 이름, 정확한 주소,
  특정 가능한 날짜·금액·얼굴·간판은 사용하지 마세요.
- 사례 게재 전 반드시 의뢰인 동의를 받아주세요.

## 7. "아무나 추가하면 안 되는" 문제 — 접근 권한 관리

- 홈페이지 방문자(고객)는 `index.html` 파일을 수정할 권한이 전혀 없습니다.
- 진짜 신경 써야 할 것은 **① 홈페이지 파일이 올라간 호스팅 계정**과
  **② 네이버 블로그 계정**, 이 두 곳의 비밀번호 관리입니다.
- 비밀번호는 변호사님(또는 신뢰하는 한 분)만 보유하시는 걸 권해드려요.
- 파일을 고치기 전에는 같은 파일을 복사해서 `index_backup.html` 같은
  이름으로 보관해두면, 실수했을 때 쉽게 되돌릴 수 있습니다.

## 8. 자주 하는 실수

- 따옴표(`"`)를 한글 입력기의 곡선 따옴표(" ")로 잘못 입력 → 오류 발생
- 마지막 사례 뒤 쉼표 누락, 또는 반대로 마지막 사례에 불필요한 쉼표 → 오류 발생
- `category`에 정해진 4개 단어 외의 다른 표현 사용 → 필터 버튼에서 안 걸러짐
- `blogUrl`에 블로그 "홈" 주소만 넣고 실제 글 주소를 안 넣음 → 클릭해도 원하는 글이 안 보임

수정 후 화면이 깨지거나 카드가 안 보이면, 방금 수정한 부분의 따옴표와 쉼표를 다시 확인해주세요.
